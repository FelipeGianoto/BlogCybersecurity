    Protocolos de Rede

  • TCP/IP: O protocolo TCP/IP é o padrão da Internet e é amplamente utilizado para comunicação em redes locais e globais. É composto por dois protocolos principais, o TCP (Transmission Control Protocol) e o IP (Internet Protocol).

  • DNS: O protocolo DNS (Domain Name System) é usado para traduzir nomes de domínio em endereços IP.

  • DHCP: O protocolo DHCP (Dynamic Host Configuration Protocol) é usado para atribuir dinamicamente endereços IP a dispositivos em uma rede.

  • FTP: O protocolo FTP (File Transfer Protocol) é usado para transferir arquivos entre computadores em uma rede.

  • HTTP: O protocolo HTTP (Hypertext Transfer Protocol) é usado para transferir páginas da web e outros conteúdos na internet.

  • SMTP: O protocolo SMTP (Simple Mail Transfer Protocol) é usado para enviar e-mails.

  • SSH: O protocolo SSH (Secure Shell) é usado para acesso remoto seguro a sistemas e dispositivos em uma rede.

  • SSL/TLS: O protocolo SSL/TLS (Secure Sockets Layer/Transport Layer Security) é usado para criptografar a comunicação entre servidores e clientes em redes, como em transações bancárias online.

    Explicação das Sete Camadas do Modelo OSI

    1. Camada Física
  • A camada física lida com a transmissão de bits brutos pela rede. Ela define as características elétricas, mecânicas e funcionais das interfaces de comunicação física, como cabos, conectores e sinais elétricos. Seu objetivo é fornecer uma conexão física confiável entre os dispositivos de rede.
    • 2. Camada de Enlace de Dados
  • A camada de enlace de dados garante a transferência confiável de dados entre dispositivos adjacentes em uma rede. Ela lida com a detecção e correção de erros na camada física, organiza os dados em quadros e controla o fluxo de dados para evitar congestionamentos.
    • 3. Camada de Rede
  • A camada de rede lida com o roteamento dos pacotes de dados pela rede. Ela determina a melhor rota para enviar os pacotes do dispositivo de origem para o destino, levando em consideração fatores como congestionamento, prioridade e endereçamento lógico dos dispositivos.
    • 4. Camada de Transporte
  • A camada de transporte fornece um transporte confiável e eficiente dos dados entre as máquinas de origem e destino. Ela divide os dados em segmentos, adiciona informações de controle, como números de sequência, para garantir a entrega adequada e gerencia o controle de fluxo e a correção de erros.
    • 5. Camada de Sessão
  • A camada de sessão estabelece, mantém e encerra as sessões entre aplicativos em dispositivos diferentes. Ela permite que aplicativos em dispositivos distintos estabeleçam uma conexão lógica e coordenem a comunicação entre si, gerenciando a troca de dados e o controle da conversação.
    • 6. Camada de Apresentação
  • A camada de apresentação lida com a representação dos dados. Ela traduz, criptografa e compacta os dados para garantir que possam ser interpretados corretamente pelo destinatário. Além disso, também lida com a compressão e decompressão de dados, caso necessário.
    • 7. Camada de Aplicação
  • A camada de aplicação é a camada mais próxima do usuário final. Ela fornece serviços de rede para os aplicativos e permite a interação direta entre o usuário e os serviços de rede. Essa camada inclui protocolos para funções como correio eletrônico (SMTP), transferência de arquivos (FTP), navegação na web (HTTP) e muitos outros.
